Notable Projects

Notemaker
Centre for the Study of Learning and Performance / Concordia University
Timeframe: 2 months training, 6 months development

Notemaker is an iOS application designed to easily allow users annotate and share media files (audio and video) between each other. It was developed collaboratively between Concordia University (CSLP), Queens University, and The Royal Conservatory of Music (RCM). Notemaker's original goal was to help music teachers leave time sensitive feedback on their students' works but due to its general purpose design it can be used for a variety of fields (ex: sports training, fitness, dance, theatre, etc.).

I worked on this project as the only software developer with the help of a designer and two instructional designers. The CSLP not having mobile application development prior to this project meant that I needed to learn everything needed to build and deploy an iOS application from scratch without guidance.

READS
Centre for the Study of Learning and Performance / Concordia University
Timeframe: 2 weeks (first release), few hours of maintenance and updates every 3~4 months

READS is a free repository of books for children. It was originally planned to be two separate components to reach both users online and offline. First, a WordPress website would be created and maintained by a (non-technical) member of the CSLP. Second, a USB flash drive would be maintained separately containing all available PDF versions of the books and an excel spreadsheet acting like an index.

When I was told of the general plan for how READS was then planned on being maintained, I suggested getting a programmer involved and making one version of READS that could work online and offline. My idea was to convert the data stored in the excel spreadsheet into JSON and load it into a static AngularJS webpage. This would allow the CSLP to maintain only one version of the data and having a UI for both online and offline usage. My recommendation was accepted and I was then tasked with completing it within a short timespan (2 weeks) due to timeframe and budget constraints.

READS has been generally well received especially in more remote places like Kenya. With that said READS is by no means perfect. There have always been plans to improve the UI, make the tool more bilingual friendly, and add a proper MySQL database for book entry but there was not enough time or budget to complete them all. Overall I'm personally very proud of this project since I was able to do so much in such a short time span.
